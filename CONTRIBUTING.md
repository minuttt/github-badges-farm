# Contributing to GitHub Badge Tools

Thank you for your interest in contributing! This project is designed for educational purposes to help people understand GitHub's achievement system.

## 🎯 Our Mission

This repository exists to:
- **Educate** users about GitHub's features and APIs
- **Promote ethical** use of automation
- **Provide tools** for legitimate badge earning
- **NOT** to game the system or spam GitHub

## 📋 Contribution Guidelines

### What We Welcome

✅ **Educational Content**
- Improved documentation
- Better explanations of badge mechanics
- API usage examples
- Ethical earning strategies

✅ **Code Improvements**
- Bug fixes
- Better error handling
- Code clarity improvements
- Security enhancements

✅ **Safety Features**
- Rate limiting improvements
- Better disclaimers
- Safety warnings
- ToS compliance checks

### What We Don't Accept

❌ **Harmful Content**
- Scripts that encourage spam
- Methods that violate GitHub ToS
- Abusive automation
- Anything that could harm the GitHub community

❌ **Unethical Practices**
- Bot networks
- Fake activity generation
- Self-starring mechanisms
- Badge farming exploits

## 🚀 How to Contribute

### 1. Fork & Clone

```bash
# Fork the repository on GitHub, then:
git clone https://github.com/YOUR_USERNAME/github-badges-farm.git
cd github-badges-farm
```

### 2. Create a Branch

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

### 3. Make Your Changes

- Follow existing code style
- Add comments to explain complex logic
- Include safety checks and rate limits
- Update documentation if needed

### 4. Test Your Changes

- Test scripts in your own test repository
- Ensure no harm to GitHub's infrastructure
- Verify rate limits are respected
- Check for edge cases

### 5. Commit & Push

```bash
git add .
git commit -m "feat: your descriptive commit message"
git push origin feature/your-feature-name
```

### 6. Open a Pull Request

- Describe what your PR does
- Explain why it's needed
- Reference any related issues
- Ensure it follows our ethical guidelines

## 📝 Code Style

### Shell Scripts
- Use bash shebang: `#!/bin/bash`
- Include comments for complex operations
- Add error handling
- Use descriptive variable names

### Python Scripts
- Follow PEP 8
- Add docstrings
- Handle exceptions properly
- Use type hints where appropriate

### Documentation
- Use clear, simple language
- Include examples
- Add warnings where needed
- Keep formatting consistent

## 🔒 Security

### Reporting Security Issues

If you find a security vulnerability:
1. **DO NOT** open a public issue
2. Email the maintainer privately
3. Include details and steps to reproduce
4. Wait for response before disclosure

### Security Best Practices

- Never commit API tokens or credentials
- Use environment variables for sensitive data
- Validate all user inputs
- Add rate limiting to API calls

## ⚖️ Ethical Standards

All contributions must:

1. **Respect GitHub's ToS**
   - No ToS violations
   - No community guideline breaches
   - No abuse of resources

2. **Be Educational**
   - Teach legitimate concepts
   - Explain "why" not just "how"
   - Promote understanding

3. **Be Responsible**
   - Include proper warnings
   - Add safety measures
   - Consider consequences

4. **Be Transparent**
   - Clear about what code does
   - Honest about limitations
   - Open about risks

## 📊 Pull Request Review Process

### What We Check

1. **Functionality** - Does it work as intended?
2. **Safety** - Is it safe for users and GitHub?
3. **Ethics** - Does it promote ethical use?
4. **Documentation** - Is it well documented?
5. **Code Quality** - Is it clean and maintainable?

### Approval Criteria

✅ Passes all safety checks
✅ Includes proper documentation
✅ Follows ethical guidelines
✅ Has no security issues
✅ Respects GitHub's ToS
